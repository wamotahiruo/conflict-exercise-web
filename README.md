# GitHub Conflict Exercise Web

GitHubでのコンフリクト体験を目的とした演習用ウェブサイトです。

## 概要

このプロジェクトは、JavaScriptプログラムをReactベースのウェブサイトに統合したものです。ユーザーが名前を入力すると、JavaScriptプログラムが自動的にメッセージを生成します。

## 機能

- **メッセージジェネレーター**: ユーザーが名前を入力すると、メッセージが自動生成されます
- **リアルタイム更新**: 入力フィールドの内容が変わると、メッセージがリアルタイムで更新されます
- **レスポンシブデザイン**: モバイルデバイスにも対応した設計になっています

## 技術スタック

- **フロントエンド**: React 19 + TypeScript
- **スタイリング**: Tailwind CSS 4
- **UI コンポーネント**: shadcn/ui
- **ルーティング**: Wouter

## プロジェクト構成

```
client/
  public/       ← 静的アセット
  src/
    pages/      ← ページレベルのコンポーネント
    components/ ← 再利用可能なUIコンポーネント
    contexts/   ← Reactコンテキスト
    hooks/      ← カスタムReactフック
    lib/        ← ユーティリティ関数
    App.tsx     ← ルーティングと最上位レイアウト
    main.tsx    ← Reactエントリーポイント
    index.css   ← グローバルスタイル
```

```bash

```

## コンフリクト演習について

このプロジェクトは、GitHubでのコンフリクト解決を体験するための演習用です。複数のブランチで異なる変更を加えることで、マージ時のコンフリクトを意図的に発生させることができます。

### 推奨される演習方法

1. **複数のブランチを作成**: `feature/branch1` と `feature/branch2` などのブランチを作成
2. **異なる変更を加える**: 各ブランチで異なる部分を編集
3. **マージしてコンフリクトを発生させる**: `git merge` コマンドでコンフリクトを発生させる
4. **コンフリクトを解決**: エディタでコンフリクトを手動で解決
